# mapp-i-journal

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存放，不针对任何特定域名或网站，仅作为通用页面存档使用。

## 项目简介

mapp-i-journal 是一个简单的静态页面集合仓库。每个 HTML 文件均为独立页面，可单独打开查看，无需额外服务端支持。适用于需要批量保存、整理或展示 HTML 内容的场景。

## 目录说明

```
mapp-i-journal/
├── index.html          # 仓库主页（可自定义）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 页面所需的资源文件（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `index.html`：仓库默认展示页面，可自行修改。
- `pages/`：所有独立 HTML 页面存放于此，建议按需命名。
- `assets/`：存放页面引用的静态资源，保持目录整洁。

## 页面归档说明

- 每个页面均为独立完整 HTML 文件，可离线打开。
- 页面间无依赖关系，便于单独使用或分享。
- 若页面引用外部资源，请确保资源路径正确或一并归档至 `assets/` 目录。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来增加、修改或删除页面。
- 请保持页面内容简洁，避免引入过多外部依赖。
- 本仓库不托管动态内容或后端服务，所有页面应为纯前端实现。
- 如有特殊排版或功能需求，请在提交时说明。
